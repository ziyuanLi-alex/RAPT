# RAPT Agent Notes

## Project Shape
- RAPT is a Python RFID collection GUI/CLI project.
- GUI entry point: `src/main.py`.
- Main window/navigation: `src/ui/main_window.py`.
- GUI views live in `src/ui/views/`.
- Long-running GUI work is handled by `QThread` subclasses in `src/ui/threads.py`.
- Core RFID streaming is in `src/core/DataCollector.py`.
- Runtime path helpers for packaged/dev mode are in `src/core/paths.py`.

## Packaging And Runtime Files
- Use `RAPT.spec` for Windows GUI builds.
- Build command: `python -m PyInstaller --noconfirm --clean RAPT.spec`.
- Expected packaged executable: `dist\RAPT\RAPT.exe`.
- Do not use stale `dist\main.exe`.
- In packaged mode, runtime `config.json`, `binding.json`, and relative output paths are resolved next to `RAPT.exe`.
- In dev mode, runtime files resolve from the repo root.

## Configuration
- `ConfigManager` is a singleton in `src/core/settings.py`.
- Persisted config fields include:
  - `baud`
  - `com`
  - `frame_duration_ms`
  - `output_dir`
  - `output_format`
  - `skellycam_base_url`
  - `skellycam_recording_dir`
- Default SkellyCam base URL: `http://localhost:53117`.
- Default SkellyCam recording directory: `H:\lib\Skellycam_recording`.

## SkellyCam Integration
- RAPT controls experiments/sessions; SkellyCam remains the camera/video authority.
- Do not import FreeMoCap or SkellyCam Python packages into RAPT.
- Use SkellyCam HTTP API only, via `src/core/skellycam_client.py`.
- Current client functions:
  - `check_health(base_url)`
  - `start_recording(base_url, recording_name, recording_directory, mic_device_index=-1)`
  - `stop_recording(base_url)`
- Start/stop requests record `time.perf_counter_ns()` before/after/midpoint and preserve response text.
- Live GUI workflow is `IntegratedInterface` in `src/ui/views/integrated_interface.py`.
- Legacy/manual OpenCV video capture remains in `VideoInterface`; keep it separate from SkellyCam integration.

## Integrated Session Data
- Integrated sessions write under:
  `config.output_dir / RAPT_dataset / session_YYYY_MM_DD_NNN /`
- Session helpers live in `src/core/integrated_session.py`.
- RFID files live under the session `rfid/` directory:
  - `rfid_reads.csv`
  - `rfid_events.csv`
- Metadata file:
  - `session_meta.json`
- SkellyCam recording files are not moved by RAPT. RAPT records the SkellyCam recording name, base URL, and recording directory in metadata.

## RFID CSV Schemas
- `rfid_reads.csv` columns:
  `session_id,trial_id,tag_id,antenna,rssi,phase,frequency,reader_timestamp,host_wall_time_ns,host_perf_counter_ns,event_type`
- `rfid_events.csv` columns:
  `session_id,trial_id,event_type,host_wall_time_ns,host_perf_counter_ns,notes`
- Raw RFID logging must stamp each read with both:
  - `time.time_ns()`
  - `time.perf_counter_ns()`
- V1 sync markers are `sync_start` and `sync_end`.
